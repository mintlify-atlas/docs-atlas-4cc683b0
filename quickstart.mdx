This guide walks you through parsing a Swift source string, printing the resulting syntax tree, and using a `SyntaxRewriter` to transform nodes. All examples use real swift-syntax APIs.

<Note>
Before you start, add `SwiftParser` and `SwiftSyntax` as dependencies. See [Installation](/installation) for instructions.
</Note>

<Steps>
  <Step title="Parse a Swift source string">
    `Parser.parse(source:)` accepts a `String` containing Swift source code and returns a `SourceFileSyntax` â€” the root node of the syntax tree. The parser always succeeds; any syntax errors are represented as missing or unexpected nodes within the tree rather than thrown errors.

    ```swift
    import SwiftParser
    import SwiftSyntax

    let source = """
    func greet(name: String) -> String {
        return "Hello, \\(name)!"
    }
    """

    let sourceFile = Parser.parse(source: source)
    print(sourceFile)
    ```

    Running this prints the source text back, because `SourceFileSyntax` conforms to `CustomStringConvertible` and produces source-accurate output.

    <Tip>
      The syntax tree is *source-accurate*: every byte of input, including whitespace and comments, is preserved as trivia attached to tokens. Printing any node always reproduces the original text for that subtree.
    </Tip>
  </Step>

  <Step title="Walk the tree with SyntaxVisitor">
    Subclass `SyntaxVisitor` and override the `visit(_:)` method for the node types you care about. Call `walker.walk(sourceFile)` to start the traversal.

    The example below collects the name of every function declared in the file:

    ```swift
    import SwiftSyntax

    class FunctionNameCollector: SyntaxVisitor {
        var names: [String] = []

        override func visit(_ node: FunctionDeclSyntax) -> SyntaxVisitorContinueKind {
            names.append(node.name.text)
            return .visitChildren
        }
    }

    let collector = FunctionNameCollector(viewMode: .sourceAccurate)
    collector.walk(sourceFile)
    print(collector.names) // ["greet"]
    ```

    `visit(_:)` returns `SyntaxVisitorContinueKind`. Return `.visitChildren` to descend into the node's children, or `.skipChildren` to stop traversal at this node.
  </Step>

  <Step title="Rewrite nodes with SyntaxRewriter">
    `SyntaxRewriter` works like `SyntaxVisitor` but returns a (potentially modified) copy of each node. Override the `visit(_:)` overload for the node type you want to transform. The rewriter rebuilds only the parts of the tree that changed.

    The example below is adapted from the `AddOneToIntegerLiterals` example in the swift-syntax repository. It increments every integer literal in a source file:

    ```swift
    import SwiftParser
    import SwiftSyntax

    class AddOneToIntegerLiterals: SyntaxRewriter {
        override func visit(_ token: TokenSyntax) -> TokenSyntax {
            // Only transform integer literal tokens.
            guard case .integerLiteral(let text) = token.tokenKind else {
                return token
            }

            // Remove underscores (e.g. 3_000 -> 3000) then parse the integer.
            let integerText = String(text.filter { ("0"..."9").contains($0) })
            guard let int = Int(integerText) else { return token }

            // Return a new token with the incremented value.
            var newToken = token
            newToken.tokenKind = .integerLiteral("\(int + 1)")
            return newToken
        }
    }

    let source = """
    let x = 2
    let y = 3_000
    """

    let tree = Parser.parse(source: source)
    let rewritten = AddOneToIntegerLiterals().visit(tree)
    print(rewritten)
    // let x = 3
    // let y = 3001
    ```

    <Note>
      `SyntaxRewriter` produces a new tree value; the original `tree` is unchanged. SwiftSyntax types are value types (backed by a copy-on-write arena), so rewriting is non-destructive.
    </Note>
  </Step>

  <Step title="Access node children and tokens">
    Every syntax node exposes its children as typed properties. You can navigate the tree directly without a visitor.

    ```swift
    import SwiftParser
    import SwiftSyntax

    let source = "func greet(name: String) -> String { return \"Hello\" }"
    let tree = Parser.parse(source: source)

    // SourceFileSyntax has a `statements` child (CodeBlockItemListSyntax).
    // Each item wraps a statement or declaration.
    for statement in tree.statements {
        if let funcDecl = statement.item.as(FunctionDeclSyntax.self) {
            print("Function name:", funcDecl.name.text)

            // Walk the parameter list.
            for param in funcDecl.signature.parameterClause.parameters {
                print("  Parameter:", param.firstName.text)
            }
        }
    }
    // Function name: greet
    //   Parameter: name
    ```

    Use `node.as(SomeType.self)` to attempt a downcast to a specific syntax type â€” it returns `nil` if the node is not of that type. Use `node.is(SomeType.self)` to check without casting.

    <Tip>
      The interactive Swift AST Explorer at [swift-ast-explorer.com](https://swift-ast-explorer.com) lets you paste any Swift source and explore the full node hierarchy in your browser â€” a fast way to discover which node types and property names to use.
    </Tip>
  </Step>
</Steps>
