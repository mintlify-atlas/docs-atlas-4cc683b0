`SwiftRefactor` provides two protocols for implementing source transformations — one for structural syntax rewrites and one for textual edits — along with a set of built-in refactoring actions.

## SyntaxRefactoringProvider

Prefer this protocol when your transformation can be expressed as replacing one syntax node with another:

```swift
public protocol SyntaxRefactoringProvider: EditRefactoringProvider {
    associatedtype Input: SyntaxProtocol
    associatedtype Output: SyntaxProtocol
    associatedtype Context = Void

    static func refactor(syntax: Input, in context: Context) throws -> Output
}
```

`SyntaxRefactoringProvider` automatically provides the `EditRefactoringProvider` implementation that converts the returned node into a `[SourceEdit]`.

### Context-free convenience

When `Context == Void`, a simpler overload is available:

```swift
static func refactor(syntax: Input) throws -> Output
```

## EditRefactoringProvider

Use this protocol when your transformation produces textual edits rather than a new node:

```swift
public protocol EditRefactoringProvider {
    associatedtype Input: SyntaxProtocol
    associatedtype Context = Void

    static func textRefactor(syntax: Input, in context: Context) throws -> [SourceEdit]
}
```

Return an empty array (`[]`) when the refactoring does not apply to the given node, rather than throwing.

## RefactoringNotApplicableError

Throw this error to signal that the refactoring cannot be applied. The message should start lowercase, following compiler error convention:

```swift
public struct RefactoringNotApplicableError: Error, CustomStringConvertible {
    public var description: String
    public init(_ reason: String)
}
```

## Implementing a refactoring action

```swift
import SwiftRefactor
import SwiftSyntax

/// Removes redundant parentheses around a return expression.
struct RemoveRedundantParens: SyntaxRefactoringProvider {
    typealias Input = TupleExprSyntax
    typealias Output = ExprSyntax

    static func refactor(syntax: TupleExprSyntax) throws -> ExprSyntax {
        // Preflight: only applies to single-element tuples with no label
        guard
            syntax.elements.count == 1,
            syntax.elements.first?.label == nil
        else {
            throw RefactoringNotApplicableError("not a redundant parenthesized expression")
        }
        return ExprSyntax(syntax.elements.first!.expression)
    }
}
```

## Built-in refactoring actions

`SwiftRefactor` ships several ready-to-use actions:

| Action | Input | Description |
|---|---|---|
| `CallToTrailingClosures` | `ExprSyntax` | Converts trailing argument closures to trailing closure syntax |
| `ConvertComputedPropertyToStored` | `VariableDeclSyntax` | Converts a computed property to a stored property |
| `ConvertStoredPropertyToComputed` | `VariableDeclSyntax` | Converts a stored property to a computed property |
| `ConvertComputedPropertyToZeroParameterFunction` | `VariableDeclSyntax` | Converts a computed property to a `func` |
| `ConvertZeroParameterFunctionToComputedProperty` | `FunctionDeclSyntax` | Converts a zero-parameter `func` to a computed property |
| `MigrateToNewIfLetSyntax` | `IfExprSyntax` | Rewrites old-style `if let x = x` to shorthand `if let x` |
| `OpaqueParameterToGeneric` | `FunctionDeclSyntax` | Converts `some Protocol` parameters to generic constraints |
| `FormatRawStringLiteral` | `StringLiteralExprSyntax` | Adds/removes raw string delimiters (`#"..."#`) |
| `AddSeparatorsToIntegerLiteral` | `IntegerLiteralExprSyntax` | Adds `_` separators for readability |
| `RemoveSeparatorsFromIntegerLiteral` | `IntegerLiteralExprSyntax` | Removes `_` separators |
| `ExpandEditorPlaceholder` | `ExprSyntax` | Expands an editor placeholder token |
